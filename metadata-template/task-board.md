# Task Board

> This file is the human-readable task backlog. The Lead reads this to stock the Beads queue.
> Format: one section per epic/feature. The Lead publishes tasks from here to Beads when dependencies are met.

## How to use this file

1. Add epics and tasks here when planning work with the Command Center.
2. Include dependency notes so the Lead knows the correct publishing order.
3. The Lead updates the status column as tasks move through the queue.
